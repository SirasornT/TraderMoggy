# MoggyTrader — Product Specification v0.1

> A personal stock research assistant that surfaces social buzz and produces
> analyst-style research memos. It describes; it does not prescribe.

---

## 1. Objective

**Primary user:** One person (the author) doing pre-trade research on individual equities.

**Problem being solved:** Manually scanning Reddit, StockTwits, financial news, charts,
and fundamentals for a ticker takes 30–60 minutes. MoggyTrader collapses that to a
single CLI command that runs all four lenses in parallel and produces a structured memo.

**Success criterion for v1:**
- `moggy discover` returns a ranked list of buzzing tickers in < 30 seconds.
- `moggy research AAPL` produces a saved markdown memo in < 90 seconds.
- The memo is accurate enough that the user would have found the same key facts
  through manual research.

## 2. Two Modes

### 2.1 Discovery Mode — `moggy discover`

Polls Reddit and StockTwits for unusual social activity across a watchlist of
symbols (or a broad scan). Returns a ranked table of tickers with their buzz scores.

**Buzz score formula (v1 — iterate later):**

```
volume_ratio   = mention_count_24h / (avg_daily_mentions_7d + 1)
velocity_ratio = (mentions_last_4h - mentions_prev_4h) / (mentions_prev_4h + 1)
sentiment      = (positive - negative) / (total_mentions + 1)   # range: [-1, 1]

buzz_score = round(
    volume_ratio
    * (1 + max(velocity_ratio, 0))
    * (1 + 0.5 * sentiment),
    2
)
```

- `volume_ratio` captures relative spike vs. the ticker's own baseline.
- `velocity_ratio` rewards tickers gaining momentum *right now* (last 4 h vs. prior 4 h).
  Negative velocity does not penalise below baseline.
- `sentiment` is a mild multiplier (±50%), not the dominant signal.

**Output:** Rich-formatted table printed to terminal, sorted descending by buzz score.

```
┌──────┬────────────┬──────────┬───────────┬───────────┬─────────────┐
│ Rank │ Ticker     │ Mentions │ Vol Ratio │ Velocity  │ Buzz Score  │
├──────┼────────────┼──────────┼───────────┼───────────┼─────────────┤
│    1 │ NVDA       │      843 │      4.2x │     +1.8x │       22.68 │
│    2 │ GME        │      312 │      2.1x │     +0.4x │        6.17 │
└──────┴────────────┴──────────┴───────────┴───────────┴─────────────┘
```

### 2.2 Deep-Dive Mode — `moggy research <TICKER>`

Runs four analyst agents concurrently, then a synthesis agent combines their
reports into a single markdown memo saved to `output/<TICKER>_<date>.md`.

**Agent pipeline (all four run in parallel via `asyncio.gather`):**

| Agent | Data source | Output |
|---|---|---|
| `SocialAgent` | Reddit (PRAW) + StockTwits REST | Sentiment summary, top posts/comments |
| `FundamentalsAgent` | yfinance | P/E, revenue growth, margins, debt/equity |
| `TechnicalAgent` | yfinance OHLCV | RSI, MACD, Bollinger Bands, trend reading |
| `NewsAgent` | yfinance `.news` | Top 10 recent headlines, macro context |

**Synthesis Agent:** Receives all four reports as context and produces the final memo.
No debate rounds. No recommendations. Describes only.

**Memo structure:**
```markdown
# Research Memo: AAPL — 2026-04-30

## TL;DR
## Social Sentiment
## Fundamentals
## Technical Picture
## Recent News
## Key Risks & Watch Items
---
*Generated by MoggyTrader. Not financial advice.*
```

## 3. CLI Commands

```bash
# Discovery
moggy discover
moggy discover --sources reddit,stocktwits   # default: both
moggy discover --limit 20                    # top N results (default: 15)
moggy discover --min-score 5.0               # filter below threshold

# Deep-dive
moggy research AAPL
moggy research AAPL --output ./reports/      # custom output dir (default: ./output/)
moggy research AAPL --model claude-opus-4-7  # override default model

# Utility
moggy config                                 # show current config (redacts secrets)
moggy version                                # print version
```

## 5. Tech Stack

| Layer | Choice | Reason |
|---|---|---|
| Language | Python 3.11+ | Stable, broad library support |
| Package manager | uv | Fast, modern, replaces pip+venv |
| CLI framework | Typer | Type-annotated, auto-generates `--help` |
| LLM | Anthropic SDK (Claude) | User's preference; supports async |
| Default model | `claude-sonnet-4-6` | Fast + capable; opus via `--model` flag |
| Social (Reddit) | PRAW | Official, well-maintained, free |
| Social (StockTwits) | httpx (REST) | Public API, no auth required |
| Market data | yfinance | Free, no key, covers OHLCV + fundamentals + news |
| Technical indicators | pandas-ta | Pure-pandas, no TA-Lib C dependency |
| Terminal output | Rich | Tables, progress spinners, markdown |
| Config/secrets | pydantic-settings + python-dotenv | Typed config, `.env` loading |
| Testing | pytest + pytest-asyncio | Standard async test support |

## 6. Configuration

All secrets and tunable defaults live in `.env`:

```env
# Required
ANTHROPIC_API_KEY=sk-ant-...

# Required for Reddit (register at reddit.com/prefs/apps — takes ~2 min)
REDDIT_CLIENT_ID=...
REDDIT_CLIENT_SECRET=...
REDDIT_USER_AGENT=MoggyTrader/0.1

# Optional tuning
DEFAULT_MODEL=claude-sonnet-4-6
DISCOVER_LIMIT=15
DISCOVER_MIN_SCORE=0.0
OUTPUT_DIR=./output
```

`.env` is gitignored. `.env.example` is committed with placeholder values.

## 7. Code Style

- **Typing:** Full type annotations on all public functions and class signatures.
- **Async:** All I/O (LLM calls, HTTP, yfinance) uses `async`/`await`. Blocking yfinance
  calls are wrapped with `asyncio.to_thread()`.
- **Error handling:** Data fetch failures return a typed `DataError` result rather than
  raising. Agents degrade gracefully — if one agent fails, the memo notes the gap
  rather than aborting.
- **No comments explaining what**: only comments explaining *why* something non-obvious
  is done.
- **Dataclasses / Pydantic models** for all inter-agent payloads (`AgentReport`,
  `BuzzScore`, `TickerData`). No raw dicts passed between layers.
- **Line length:** 100 chars. Formatter: `ruff format`. Linter: `ruff check`.

## 8. Testing Strategy

| Layer | Approach |
|---|---|
| Buzz scorer | Pure unit tests — no I/O, formula is deterministic |
| Data wrappers | Unit tests with `pytest-mock` or `respx` to mock HTTP/yfinance |
| Agents | Unit tests with mocked LLM responses (no live API calls in CI) |
| Pipeline | Integration test with a fixed ticker (AAPL) against live APIs — run manually, not in CI |

CI runs `pytest tests/unit/` only. Integration tests are opt-in: `pytest tests/integration/ -m integration`.

## 9. Boundaries

### Always do
- Append `*Not financial advice.*` to every generated memo.
- Timestamp every output file.
- Log which data sources were unavailable so the user knows gaps exist.
- Respect rate limits: add jitter to Reddit/StockTwits polling; cache yfinance responses
  within a session.

### Ask before doing
- Changing the buzz score formula.
- Adding a new data source.
- Changing the memo structure significantly.

### Never do
- Place, simulate, or suggest orders.
- Store API keys anywhere other than `.env` (never hardcode, never log).
- Make autonomous "buy / sell / hold" statements in any output.
- Add a web UI, broker integration, or backtesting framework (these are explicit non-goals).
